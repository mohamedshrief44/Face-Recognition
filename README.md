## Overview

This project implements face recognition on the **ORL (AT&T) Face Database** (400 images, 40 subjects × 10 images each) using classical dimensionality reduction paired with K-Nearest Neighbour classification.

Three pipelines are benchmarked:
- **PCA** with varying variance retention thresholds
- **Multiclass LDA** with PCA pre-processing to address the Small Sample Size problem
- **Binary LDA** for face vs. non-face classification

---

## Algorithms

### PCA — Principal Component Analysis
- Economy SVD on the 200×200 covariance matrix (avoids full 10304×10304 computation)
- Variance retention threshold α ∈ {0.80, 0.85, 0.90, 0.95}
- Number of retained components `r` determined automatically per threshold

### Multiclass LDA — Linear Discriminant Analysis
- PCA pre-processing to resolve the Small Sample Size (SSS) problem
- Within-class scatter regularisation: Sᵥ + λI, λ = 1e-4
- 39 discriminant components (C−1 for 40 classes)

### Binary LDA — Face vs. Non-Face
- Single Fisher eigenvector (rank(Sᵦ) = 1 for binary case)
- Non-face samples generated by random permutation of face images

### K-NN Classifier
- K ∈ {1, 3, 5, 7}
- Euclidean distance in the projected subspace
- Tie-breaking by minimum mean distance

## Results

| Method | Train/Test Split | K | Accuracy |
|---|:---:|:---:|:---:|
| PCA α=0.80 (r=36) | 5/5 | 1 | **95.00%** |
| Multiclass LDA | 5/5 | 1 | 59.50% |
| PCA α=0.85 (r=57) | 7/3 | 1 | **96.67%** |
| Multiclass LDA | 7/3 | 1 | **82.50%** |
| Binary LDA (face / non-face) | — | 1 | **97–98%** |

---

## Key Insights

- **Higher α hurts PCA accuracy** — retaining more variance captures noise rather than signal. The sweet spot is α = 0.80–0.85.
- **LDA is data-hungry** — with only 5 training samples per class, LDA scores 59.50%. Increasing to 7 samples yields a +23% jump, confirming the bottleneck is data volume, not the method.
- **Optimal K differs by method** — K=1 is best for PCA's clean projections; K=3 gives a marginal edge for LDA due to noisier subspace geometry.
- **Binary LDA is highly effective** — achieving 97–98% on the face/non-face task with a single discriminant component.

## Dataset Setup

The dataset is **not included** in this repository.

**Download:** [ORL / AT&T Face Database](https://cam-orl.co.uk/facedatabase.html)

After downloading, extract and place the data under the following structure:

```
data/
└── orl_faces/
    ├── s1/        # 10 images: 1.pgm … 10.pgm
    ├── s2/
    ├── s3/
    └── ...
    └── s40/
```

**Dataset specs:**
- 400 images total
- 40 subjects × 10 images each
- 92 × 112 px grayscale PGM format
- 10,304 features per image (flattened)
